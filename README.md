# Recipe App

A full-stack recipe management application with a **Spring Boot** REST API and an **Angular** single-page frontend. Users can view, search, add, edit, and delete recipes.

## Tech Stack

| Layer    | Technology |
|----------|------------|
| Backend  | Java 21, Spring Boot 4.1, Spring Data JPA, Spring Security, Hibernate Validation |
| Database | H2 (in-memory, default), PostgreSQL (optional) |
| Frontend | Angular 20, RxJS, Reactive Forms |
| Testing  | JUnit 5, MockMvc, Jasmine, Karma |

## Project Structure

```
recipe-app/
├── backend/                 # Spring Boot REST API
│   ├── src/main/java/com/example/demo/
│   │   ├── controller/      # REST endpoints
│   │   ├── model/           # JPA entities
│   │   ├── repository/      # Data access
│   │   └── config/          # Security, seed data
│   └── src/main/resources/
│       └── application.properties
├── frontend/                # Angular SPA
│   ├── src/app/
│   │   ├── components/      # Recipe list & form UI
│   │   ├── services/        # HTTP API client
│   │   ├── models/          # TypeScript interfaces
│   │   ├── interceptors/    # Basic auth for write requests
│   │   └── environments/    # Environment-specific config
│   └── proxy.conf.js        # Dev proxy to backend
└── README.md
```

## Features

- **CRUD operations** for recipes (name, ingredients, instructions)
- **Search** recipes by name
- **Loading states** on add, update, and delete actions
- **Input validation** on frontend and backend
- **HTTP Basic Authentication** for write operations (POST, PUT, DELETE)
- **Sample data** seeded on first startup
- **Unit tests** for controller and service layers

## Prerequisites

- **Java 21+**
- **Maven 3.9+** (or use the Maven wrapper if configured)
- **Node.js 20+** and **npm**
- **Angular CLI** (installed via `npm install` in `frontend/`)

## Getting Started

### 1. Clone and configure

```bash
git clone <repository-url>
cd recipe-app
```

**Backend (optional local overrides):**

```bash
cp backend/application-local.properties.example backend/application-local.properties
```

**Frontend:**

```bash
cd frontend
npm install
cp .env.example .env
```

Keep frontend auth credentials in sync with the backend defaults in `application.properties`.

### 2. Start the backend

```bash
cd backend
mvn spring-boot:run
```

The API runs at **http://localhost:8080**.

On first startup, three sample recipes are inserted automatically.

### 3. Start the frontend

```bash
cd frontend
npm start
```

The app runs at **http://localhost:4200** and proxies `/api` requests to the backend.

## Configuration

### Backend (`application.properties`)

| Variable | Default | Description |
|----------|---------|-------------|
| `DATABASE_URL` | `jdbc:h2:mem:testdb` | JDBC connection URL |
| `DATABASE_USERNAME` | _(empty)_ | Database username |
| `DATABASE_PASSWORD` | _(empty)_ | Database password |
| `APP_SECURITY_USER` | `chef` | Basic auth username |
| `APP_SECURITY_PASSWORD` | `recipe123` | Basic auth password |
| `H2_CONSOLE_ENABLED` | `true` | Enable H2 console at `/h2-console` |
| `JPA_DDL_AUTO` | `update` | Hibernate schema strategy |

Example override at runtime:

```bash
export APP_SECURITY_PASSWORD=my-secret
export DATABASE_URL=jdbc:postgresql://localhost:5432/recipedb
mvn spring-boot:run
```

### Frontend

| File | Purpose |
|------|---------|
| `frontend/.env` | Dev proxy target (`API_URL`) |
| `frontend/src/environments/environment.ts` | API URL and auth credentials (dev) |
| `frontend/src/environments/environment.prod.ts` | Production config |

Update `environment.ts` auth values to match your backend credentials.

## API Reference

Base URL: `http://localhost:8080/api/recipes`

| Method | Endpoint | Auth Required | Description |
|--------|----------|---------------|-------------|
| `GET` | `/api/recipes` | No | List all recipes |
| `POST` | `/api/recipes` | Yes | Create a recipe |
| `PUT` | `/api/recipes/{id}` | Yes | Update a recipe |
| `DELETE` | `/api/recipes/{id}` | Yes | Delete a recipe |

### Recipe JSON

```json
{
  "id": 1,
  "name": "Fluffy Pancakes",
  "ingredients": ["flour", "eggs", "milk"],
  "instructions": "Mix and cook on a griddle until golden."
}
```

`id` is generated by the server on create. `name` and `instructions` cannot be blank.

### Example requests

**List recipes (no auth):**

```bash
curl http://localhost:8080/api/recipes
```

**Create recipe (auth required):**

```bash
curl -X POST http://localhost:8080/api/recipes \
  -u chef:recipe123 \
  -H "Content-Type: application/json" \
  -d '{"name":"Test","ingredients":["flour"],"instructions":"Bake"}'
```

**Unauthenticated write (returns 401):**

```bash
curl -i -X POST http://localhost:8080/api/recipes \
  -H "Content-Type: application/json" \
  -d '{"name":"Test","ingredients":["flour"],"instructions":"Bake"}'
```

## Authentication

Write operations are protected with **HTTP Basic Authentication**.

- **Public:** `GET /api/recipes`
- **Protected:** `POST`, `PUT`, `DELETE`

The Angular app sends credentials automatically via an HTTP interceptor for mutating requests. Default demo credentials:

- Username: `chef`
- Password: `recipe123`

> **Note:** This is a learning setup. Use proper user management and token-based auth for production.

## Building for Production

### Backend

```bash
cd backend
mvn clean package
java -jar target/demo-0.0.1-SNAPSHOT.jar
```

### Frontend

```bash
cd frontend
npm run build
```

Output is written to `frontend/dist/recipe-app/`. Serve the static files with any web server and configure the API URL for your deployment environment.

## Development Notes

- The frontend dev server uses `proxy.conf.js` to forward `/api` to `http://localhost:8080`.
- H2 is in-memory by default; data is reset when the backend restarts.
- PostgreSQL support is included in the backend POM for future production use.
- After changing backend security settings, restart the Spring Boot application.
